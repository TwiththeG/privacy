Privacy Policy

Sentra DLP — Chrome Extension

Last updated: April 2026

Overview

Sentra DLP is a browser extension for privacy protection that detects and blocks data leaks, trackers, and digital fingerprinting on websites. This policy explains what data we access, how we use it, and your rights.

✓  We do not sell, rent, or share your personal data with any third party for commercial purposes.

Data We Access

  • Current tab URL — accessed only when you explicitly click "Report URL". Used solely to submit a threat report to the backend server.
  • Security event metadata — type of threat detected (e.g., WebRTC leak, canvas fingerprinting), severity level, and timestamp. Never includes browsing history, passwords, or form data.
  • Extension settings — your feature preferences (which protections are enabled/disabled), language choice, and bypass list. Stored locally on your device.

Data We DO NOT Collect

  • Passwords or authentication credentials
  • Form inputs or personal data you type on websites
  • Complete browsing history
  • Financial information
  • Your name, email address, or any identifying personal information
  • Location data

How Data Is Used

Security event metadata is optionally transmitted to the Sentra DLP backend server for:

  • Displaying your personal security dashboard at the Dashboard
  • Generating aggregated threat statistics
  • Sending high-severity alerts via Telegram (if configured by the system administrator)

The Report URL feature requires your explicit action (button click) and explicit consent before transmitting any URL data.

Data Storage

All extension preferences and local event logs are stored using chrome.storage.local on your device only. No data is sent to any server without your knowledge.

The backend server retains event logs for operational purposes only and does not link them to any personal identity.

Third-Party Services

  • Telegram Bot API — used only for high-severity security alerts when explicitly configured. No personally identifiable information is transmitted.
  • Google Fonts — used in the dashboard UI. Subject to Google's own privacy policy.

Permissions Justification

  • activeTab / scripting — required to inject the protection overlay UI into web pages
  • storage — required to save your preferences locally
  • declarativeNetRequest — required to block tracker and malicious network requests
  • privacy — required to disable WebRTC IP leaks at the browser level
  • alarms — required for periodic sync of security rules

Your Rights & Controls

  • You can disable any protection feature from the extension popup at any time
  • You can add websites to the bypass list to exclude them from monitoring
  • You can uninstall the extension at any time to remove all locally stored data
  • The Report URL feature requires explicit per-session consent

Academic Context
